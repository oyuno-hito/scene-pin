# パーソナル動画ブックマーク＆プレイヤー 要求仕様書（フロントエンド完結版）

## 1. 概要
本プロジェクトは、iOSデバイス（Safari）上で動作するブラウザベースの動画視聴ツールである。外部サーバーを構築せず、ブラウザのローカルストレージを活用することで、メンテナンスコストを抑えつつ「動画の特定箇所への瞬時ジャンプ・管理」を実現する。

## 2. 機能要件

### 2.1 動画再生機能
* **ファイル読み込み:** iOS「ファイル」アプリやiCloud Driveから動画を選択・再生する。
* **再生制御:** 再生/一時停止、シーク、ボリューム調整。
* **再生速度調整:** `playbackRate` をUIから動的に変更可能（0.5x 〜 2.0x等）。

### 2.2 ブックマーク（ピン）機能
* **タイムスタンプ保存:** 再生中の任意位置にマーカー（ピン）を打つ。
* **編集機能:** 保存したタイムスタンプに対するメモの編集・削除。
* **ジャンプ機能:** リスト化されたタイムスタンプをタップし、該当箇所へ瞬時にシーク。

### 2.3 ループ再生機能
* **A-B間ループ:** 指定した開始点と終了点の間をリピート再生する。

## 3. 非機能要件
* **プラットフォーム:** iOS Safari（PWAとしてホーム画面に追加し、アプリライクに動作させる）。
* **データ永続化:** サーバーは使用せず、`IndexedDB` または `LocalStorage` を使用してデバイス内にデータを保存する。
* **通信仕様:** 外部通信は発生させない（プライバシー・セキュリティの担保）。
* **動画編集:** **対象外**（編集は標準アプリで行い、本アプリはビューアーに特化する）。

## 4. 技術スタック案
* **フレームワーク:** React / Vue.js / Svelte などのモダンフレームワーク（SPA構成）。
* **ビデオライブラリ:** `video.js` または `Plyr`（ブラウザ標準の `video` タグを制御）。
* **データ管理:** `IndexedDB`（`Dexie.js` 等のラッパーライブラリを併用すると実装が容易）。
* **PWA化:** `manifest.json` と Service Worker（オフライン動作・ホーム画面インストール対応）。

## 6. 運用・拡張方針
* サーバーレスであるため、GitHub Pages や Vercel などの静的ホスティングサービスにデプロイするだけで、PCからもスマホからもアクセス可能な自分専用アプリとして即座に運用開始が可能。
* 将来的にサーバー同期が必要になった際は、バックエンド（Node.js等）を別途作成し、クライアント側の `IndexedDB` と API を同期させるアーキテクチャに移行可能。
