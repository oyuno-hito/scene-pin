# 要求仕様書の作成

## 配置場所

```
docs/要求仕様書/v{major}.{minor}.{patch}.md
```

## フロントマター

```yaml
---
version: X.X.X
status: planned  # planned / in-progress / released
---
```

### ステータスの意味

| status | 意味 | 使用タイミング |
|--------|------|---------------|
| `planned` | 計画段階、未着手 | 仕様書作成時 |
| `in-progress` | 現在実装中 | 実装開始時に変更 |
| `released` | 実装済み | 実装完了時に変更 |

---

## バージョニング規則

| 変更の種類 | バージョン | 例 |
|-----------|-----------|-----|
| 破壊的変更、大規模リファクタ | メジャー (X.0.0) | v2.0.0 |
| 新機能追加 | マイナー (X.Y.0) | v1.1.0 |
| バグ修正、小さな改善 | パッチ (X.Y.Z) | v1.0.1 |

---

## 作成フロー

1. **既存仕様書の確認**
   - `docs/要求仕様書/` 配下を確認
   - 最新バージョンを把握

2. **バージョン番号の決定**
   - 変更の規模に応じてバージョンを決定

3. **テンプレートから作成**
   - 不要なセクションは削除してOK
   - プロジェクトに合わせてカスタマイズ

4. **ユーザーと内容を確認**
   - 仕様書のドラフトを提示
   - フィードバックを反映

5. **statusをplannedで保存**
   - 実装開始時に `in-progress` に変更
